DOMAIN 1 — AI & ML Fundamentals (20%)

Core hierarchy keywords: AI ⊃ ML ⊃ Deep Learning ⊃ Generative AI. - AI = machines doing human-like tasks. ML = learns from data, no explicit rules. Deep Learning = neural networks, many layers. GenAI = creates new content.

Learning types (high-yield):

Type Keyword Use case
Supervised labeled data classification, regression (spam filter, price prediction)
Unsupervised unlabeled, find patterns clustering, anomaly detection (customer segments)
Reinforcement reward/penalty, trial & error robotics, game AI, recommendation tuning
Semi-supervised mix labeled + unlabeled when labels are expensive

Must-know terms: - Training = teach the model on data. Inference = use trained model to predict. - Classification = predict a category. Regression = predict a number. - Features = input variables. Labels = the answers (supervised). - Overfitting = memorized training data, fails on new data. Underfitting = too simple, misses patterns. - Bias–variance tradeoff: high bias = underfit; high variance = overfit. - Structured data (tables) vs unstructured (text, images, audio). - Inferencing types: batch (large datasets, not time-sensitive) vs real-time (low latency, live).

Tip: Exam loves "Is ML the right approach?" — ML fits when you have lots of data + patterns too complex for rules. If a simple if-then rule solves it, ML is overkill (wrong answer).

DOMAIN 2 — Generative AI Fundamentals (24%)

Keywords: Foundation Model (FM), Large Language Model (LLM), transformer, tokens, embeddings, prompt, hallucination, multimodal.

  • Foundation Model (FM) = huge model pre-trained on broad data, adaptable to many tasks.
  • LLM = FM specialized for text. Transformer = the architecture behind LLMs (self-attention).
  • Tokens = chunks of text the model processes (you're billed per token).
  • Embeddings = numeric vector representation of meaning → enables semantic search & RAG.
  • Prompt = your input/instruction. Context window = how much text the model can consider at once.
  • Hallucination = confident but false output (a veracity/accuracy problem).
  • Temperature = randomness/creativity dial (low = focused, high = creative).
  • Multimodal = handles text + image + audio + video.

GenAI capabilities to recognize: text generation, summarization, translation, chatbots, code generation, image generation, search.

Benefits/limitations (tested): - ✅ Benefits: fast, adaptable, low barrier, creative, simplifies tasks. - ⚠️ Limitations: hallucinations, nondeterminism, no inherent factual guarantee, can be costly, data cutoff (no recent knowledge unless grounded).

Tip: Know why GenAI fits or doesn't. Use GenAI for content creation, summarization, conversation. Do not use it where you need guaranteed-correct numeric/factual output without grounding — that's a hallucination trap answer.

RAG vs Fine-tuning vs Prompt Engineering (VERY high-yield decision)

Need Choose
Inject current/proprietary facts, reduce hallucination, no retraining RAG (Knowledge Bases)
Change model style/behavior/format for a specialized task Fine-tuning (needs labeled data, more cost)
Quick improvement, no data, no cost Prompt engineering

Prompt engineering keywords: zero-shot (no examples), few-shot (give examples), chain-of-thought (show reasoning steps). Prompt injection / jailbreaking = attacks Guardrails defend against.

Other Domain-3 services

  • Amazon Q Developer = AI coding assistant (formerly CodeWhisperer); also helps with AWS console/resources.
  • SageMaker JumpStart = hub of pre-built FMs & solution templates to deploy quickly.
  • PartyRock = no-code Bedrock playground for experimentation.

Tip: "Reduce hallucinations / use latest internal data" → Knowledge Bases (RAG), not fine-tuning. "Block toxic output or hide PII" → Guardrails. "Take actions / call systems" → Agents. These three are the most-tested Bedrock distinctions.

DOMAIN 4 — Responsible AI (14%)

The dimensions of responsible AI (memorize the list): - Fairness (no bias across groups) · Explainability (understand why) · Transparency · Robustness · Privacy & Security · Safety · Veracity (accuracy/truthfulness) · Controllability · Governance.

Key terms: - Bias = unfair skew in data or output. Fairness = equitable treatment. - Explainability / interpretability = ability to explain a model's decision (important for regulated industries). - Transparency = being open about how AI works and its limits.

Tools that enforce responsible AI:

Tool Role
SageMaker Clarify Detect bias in data/models + explainability (feature importance)
SageMaker Model Monitor Detect drift in production (data/quality changes over time)
Bedrock Guardrails Block harmful content, denied topics, PII redaction, hallucination/contextual-grounding checks
AWS AI Service Cards Docs on intended use, limitations, fairness of AWS AI services (transparency)
SageMaker Model Cards Document a model's purpose, performance, risks (governance)
Augmented AI (A2I) Human-in-the-loop review of risky/low-confidence predictions

Tip: Match the dimension to the tool: bias/explainability → Clarify; drift → Model Monitor; toxic output/PII in GenAI → Guardrails; transparency about a service → AI Service Cards; human oversight → A2I.

AWS Artifact on-demand compliance reports (SOC, ISO) proof of compliance
PrivateLink / VPC private network access keep AI traffic off the public internet

Data protection (high-yield)

  • Encryption at rest = data stored encrypted (S3, EBS, SageMaker volumes) → use KMS.
  • Encryption in transit = data moving over the network encrypted → TLS/SSL.
  • Data residency / sovereignty = keep data in a specific AWS Region for legal reasons.
  • Bedrock privacy: your prompts and data are not used to train the base foundation models, and are not shared with model providers.
  • Anonymization / PII handling: redact PII before training/inference → Comprehend (PII detection), Macie (find PII in S3), Bedrock Guardrails (redact PII in GenAI).

AI/ML-specific security threats (the exam tests these)

Threat What it is Defense
Prompt injection Malicious input overrides the model's instructions Bedrock Guardrails, input validation
Jailbreaking Tricking the model into bypassing safety rules Guardrails, denied topics
Data poisoning Corrupting training data to bias the model data validation, trusted sources, lineage
Model inversion / extraction Attacker reconstructs training data or copies the model access control (IAM), rate limiting, monitoring
Sensitive data leakage Model reveals PII in its output Guardrails PII redaction, data minimization
Hallucination Confident false output (integrity risk) RAG / Knowledge Bases, Guardrails grounding checks

Compliance frameworks (recognize the names)

  • GDPR (EU privacy), HIPAA (US health data), SOC 1/2/3, ISO 27001, PCI DSS (payment cards).
  • Download proof of AWS compliance → AWS Artifact. Automate audit evidence → Audit Manager.

Governance concepts

Data lineage (track data origin/flow), data cataloging, model versioning & Model Cards (document purpose, risks, performance), access logging, ongoing monitoring, and bias/drift review.

Shared Responsibility Model: AWS secures the cloud infrastructure (hardware, regions, managed-service backbone); you secure your data, IAM access, encryption settings, and configuration. For managed AI services (Bedrock, Comprehend, etc.) AWS handles more; for SageMaker you manage more of the stack.

Tip: "Who called the API and when?" → CloudTrail (audit). "Monitor performance/health" → CloudWatch. "Find sensitive data in storage" → Macie. "Encrypt at rest" → KMS. "Malicious prompt / toxic output / leak PII in GenAI" → Guardrails. "Proof of compliance to give an auditor" → AWS Artifact. Don't mix up CloudTrail (audit) vs CloudWatch (performance) — that's the #1 security trap.

☁️ AWS CLOUD FUNDAMENTALS (cloud-base knowledge on the exam)

Global infrastructure - Region = geographic area (e.g., Singapore ap-southeast-1); choose for latency, cost, and compliance/data residency. - Availability Zone (AZ) = isolated data center(s) inside a Region; using multiple AZs = high availability. - Edge Location = CloudFront cache points near users for low latency.

Core services that support AI workloads - Amazon S3 = object storage → default home for training data, datasets, model artifacts. - Amazon EC2 = virtual servers; GPU instances (P/G families) for training & inference. - AWS Lambda = serverless functions; run code with no servers (e.g., trigger inference). - Amazon VPC = your private network; PrivateLink / VPC endpoints keep AI traffic off the public internet. - Vector database (Amazon OpenSearch, Aurora/RDS pgvector) = stores embeddings for RAG retrieval. - RDS / DynamoDB = relational / NoSQL databases for app data.

Ways to access AWS - Management Console (web UI) · CLI (command line) · SDK (in code) · APIs.

Pricing & cost (high-yield) - Pay-as-you-go — pay only for what you use, no upfront commitment. - On-demand vs provisioned throughput (Bedrock: provisioned = reserved capacity for steady high volume). - GenAI is billed per token (input + output); larger models and longer context cost more. - Cost levers: pick a smaller/cheaper model when it's enough, use RAG instead of fine-tuning, batch requests, cache. - AWS Pricing Calculator estimates cost; Cost Explorer / Budgets track and alert on spend.

AWS Well-Architected Framework — 6 pillars (recognize them) - Operational Excellence · Security · Reliability · Performance Efficiency · Cost Optimization · Sustainability.

Shared Responsibility (recap): AWS = security of the cloud (infrastructure); you = security in the cloud (data, access, config).

🧠 EXTRA EXAM KNOWLEDGE (high-yield odds & ends)

ML development lifecycle (know the order) - Business problem → data collection → data preparation → feature engineering → model training → evaluation → deployment → monitoring.

Model evaluation metrics - Classification: accuracy, precision, recall, F1 score, confusion matrix, AUC-ROC. - Regression: MAE, MSE, RMSE. - LLM / GenAI: BLEU (translation), ROUGE (summarization), perplexity, plus human evaluation. - Bedrock Model Evaluation runs automatic + human evaluation to compare foundation models.

Precision vs recall (common trap) - Precision = of predicted positives, how many were right (minimize false positives). - Recall = of actual positives, how many were caught (minimize false negatives).

Fit & error terms - Overfitting = memorizes training data, fails on new data. Underfitting = too simple, misses patterns. - Bias = systematic error/unfairness. Variance = over-sensitivity to training data.

Inference options - Real-time (low-latency endpoint) · batch (bulk, cheaper, not time-sensitive) · serverless · asynchronous (large payloads).

Prompt engineering quick tips - Be specific, give context and examples (few-shot), specify the output format, set a role/persona, and tune temperature for creativity.

RAG vs Fine-tuning vs Prompt engineering (one-line recap) - Need current/private facts → RAG. Change behavior/style → fine-tune. Quick win, no data/cost → prompt engineering.

Responsible AI dimensions (recap): fairness · explainability · transparency · veracity · privacy & security · safety · controllability · governance.

📖 EXAM VOCABULARY GLOSSARY (terms that show up in questions)

These are the words AIF-C01 questions love to hide behind. Know the one-line meaning of each.

Data & governance terms

  • Data literacy — the ability to read, understand, and work with data to make decisions.
  • Data governance — policies and processes for managing data quality, security, access, and ownership.
  • Data lineage — tracking where data came from and how it moved/transformed over time.
  • Data quality — how accurate, complete, consistent, and reliable the data is.
  • Data labeling / annotation — tagging data with the correct answers for supervised learning.
  • Data augmentation — creating more training data by modifying existing samples (rotate, crop, paraphrase).
  • Ground truth — the verified correct answers/labels used to train or evaluate a model.
  • Structured data — organized in tables/rows (databases). Unstructured data — text, images, audio, video.
  • Data lake — store raw data of any type at scale. Data warehouse — store structured, query-ready data.
  • ETL — Extract, Transform, Load: moving and preparing data for use.
  • Bias (data) — unrepresentative or skewed data that leads to unfair outcomes.

Drift & monitoring terms

  • Data drift — the input data distribution changes over time vs training data.
  • Concept drift — the relationship between inputs and the correct output changes over time.
  • Model decay / degradation — accuracy drops over time → needs retraining (detected by Model Monitor).

Core ML terms

  • Algorithm — the method/procedure used to learn from data. Model — the trained result of that algorithm.
  • Parameters / weights — values the model learns during training.
  • Hyperparameters — settings you configure before training (e.g., learning rate, epochs).
  • Epoch — one full pass through the training data.
  • Feature — an input variable. Label — the target answer (supervised).
  • Generalization — how well a model performs on new, unseen data.
  • Inference — using a trained model to make predictions.
  • Training / validation / test split — separate data used to train, tune, and finally evaluate a model.

Generative AI / LLM terms

  • Foundation model (FM) — large pre-trained model adaptable to many tasks.
  • Token — a chunk of text the model processes; billing unit for GenAI.
  • Embedding — a numeric vector that captures the meaning of text/data; enables semantic search & RAG.
  • Vector database — stores embeddings for fast similarity search (powers RAG).
  • Prompt — your instruction/input. Prompt engineering — crafting prompts for better output.
  • Context window — how much text the model can consider at once.
  • Temperature / Top-p / Top-k — inference settings controlling randomness/creativity of output.
  • Grounding — anchoring a model's answers in real, retrieved, factual data (reduces hallucination).
  • Hallucination — confident but false or made-up output.
  • RAG (Retrieval-Augmented Generation) — fetch relevant data and feed it to the model for accurate answers.
  • Fine-tuning — further training an FM on your labeled data to change its behavior.
  • Zero-shot / one-shot / few-shot — giving the model zero, one, or a few examples in the prompt.
  • Chain-of-thought — prompting the model to show step-by-step reasoning.
  • Multimodal — handles multiple input types (text + image + audio).
  • Transformer — the neural-network architecture behind modern LLMs.
  • Diffusion model — generates images from noise (e.g., Stability AI). GAN — generative adversarial network.
  • Agent — an AI that reasons, plans, and takes actions using tools/APIs.

Responsible AI terms

  • Fairness — treating all groups equitably, without bias.
  • Explainability / interpretability — being able to understand why a model made a decision.
  • Transparency — being open about how the AI works and its limitations.
  • Veracity — truthfulness/accuracy of output.
  • Robustness — performing reliably even with unexpected or adversarial input.
  • Toxicity — harmful, offensive, or inappropriate content.
  • Human-in-the-loop — humans review or approve AI decisions (A2I).
  • Accountability — clear ownership/responsibility for AI outcomes.
  • Controllability — ability to guide and constrain model behavior.

Business & operational terms

  • MLOps — practices to deploy, monitor, and maintain ML models in production.
  • ROI — return on investment (business value vs cost).
  • Latency — delay before a response. Throughput — volume processed per unit time.
  • Scalability — ability to handle growing load. Availability — uptime/reliability.
  • SLA — service-level agreement (promised performance/uptime).
  • POC — proof of concept (small test before full rollout).
  • Time to market — how fast you can launch (managed services = faster).

Performance metrics (quick recall)

  • Accuracy — overall correct predictions. Precision — correct positives among predicted positives.
  • Recall — correct positives among actual positives. F1 — balance of precision & recall.
  • AUC-ROC — classifier quality across thresholds. Confusion matrix — table of TP/FP/TN/FN.
  • False positive — flagged wrongly. False negative — missed wrongly.
  • BLEU (translation) · ROUGE (summarization) · Perplexity — LLM quality metrics.

🎯 FINAL EXAM-DAY TIPS

  1. Read for the "best" answer. Multiple options are often technically valid — pick the one that best fits cost, simplicity, or the exact need. AWS usually rewards the managed / least-effort option.
  2. AI service vs SageMaker vs Bedrock is the core decision. Pre-trained API → AI service. Custom model → SageMaker. GenAI/LLM → Bedrock.
  3. RAG ≠ fine-tuning. "Use latest/private facts, reduce hallucination" = RAG. "Change behavior/style" = fine-tuning.
  4. CloudTrail = audit, CloudWatch = performance. Don't swap them.
  5. Clarify = bias/explainability, Model Monitor = drift, Guardrails = GenAI safety.
  6. Responsible AI dimensions (fairness, explainability, transparency, safety, veracity, privacy, governance) appear as scenario keywords — match them to tools.
  7. When a question says "no ML expertise" or "business users" → think Canvas, Q, or a pre-built AI service, not raw SageMaker.
  8. Eliminate distractors: if a non-AI service appears (e.g., Lambda, EC2) where a purpose-built AI service exists, the AI service is usually correct.
  9. Manage time: ~80 sec/question. Flag and move on; never get stuck.
  10. Practice scenario questions (Tutorials Dojo / official AWS Skill Builder practice) — the exam style is scenario-based, and mock exams are the single biggest score-mover.
